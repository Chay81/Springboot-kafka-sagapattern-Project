We are **removing** these from `customer-service`:

* `JwtAuthFilter.java`
* `SecurityConfig.java`
* `CustomerUserDetailsService.java`

Because we are now:

**Validating JWT centrally at the Gateway**
**Allowing only `/createCustomer` and `/login` to be public**

### 1. `JwtAuthFilter.java`

> **Not needed in gateway** as-is, because it’s servlet-based (`OncePerRequestFilter`)

Instead, you already implemented a **WebFlux-compliant `JwtGatewayAuthFilter.java`** in the Gateway using `GlobalFilter`.
This replaces the old `JwtAuthFilter.java`.

**So: Don’t move it — you already replaced it with the WebFlux version.**

2. `CustomerUserDetailsService.java`

> This is only used when:

* You're doing `AuthenticationManager`-based login (e.g., using `UsernamePasswordAuthenticationToken`)
* You’re using `SecurityConfig` to wire login authentication logic

But now:

* You're directly validating email + password in your `AuthController` (`customer-service`)
* You no longer use Spring Security’s default login manager

So again: Not needed. You can safely delete it.
