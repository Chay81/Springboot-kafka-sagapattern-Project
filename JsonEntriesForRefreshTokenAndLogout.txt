Important Note:
--------------

In order to create the /login, /refreshToken and /customer/logout endpoints, we used RSA instead of AES, due to which we RSAEncrytor class and RefreshTokenScheduler for which we need place '@EnableScheduling' annotation in the main class for scheduling.

For the RSAEncrytor class we need to create a public and private key, move them under src/main/resources and rename as shown below. Also the commands to create the public and private keys is given below.

# Generate 2048-bit private key
openssl genpkey -algorithm RSA -out private.pem -pkeyopt rsa_keygen_bits:2048

# Extract the public key from the private key
openssl rsa -pubout -in private.pem -out public.pem

src/main/resources/
├── public.pem
└── private.pem

After placing them under resources we need to declare them in application.yml as shown below.
+rsa:
+  public-key: classpath:public_key.pem
+  private-key: classpath:private_key.pem

2. Post Request for /customer/logout
------------------------------------

The json body for this endpoint will be refreshToken, as explained above.

If we use an invalid refreshtoken then error message in postman will be shown as refreshToken not found, we need to understand that token can be refreshed, but the refreshToken generated during /login will be same till logout.

Once we use the jsonbody, and the token in Authorization tab, will be logged out from the session.

We can check the same in the DB of refreshToken table in the boolean column which is expires will shown to 'true'. 

The details can be found in the localhost:8083/admin/tokens/expired, this is a Get request. We need to use the same token which was generated from /login endpoint. Below is the response in postman.
