JsonEntriesforTesting in Normal scenario

POST Request to create an order
endpoint: localhost:8081/order-service/orders
if spring gateway is used
then the endpoint will be localhost:8765/orders

{
  "productName": "tables",
  "quantity": 20,
  "price": 2,
  "status":"created"
}

Response in postman:
{
    "message": "Order placed and sent to Kafka!",
    "order": {
        "orderId": 34,
        "productName": "tables",
        "quantity": 2,
        "price": 20000.0,
        "status": "created"
    }
}

Get Request to fetch order by id
Example: if the order id is 34 as shown above
then endpoint will be localhost:8081/order-service/orders/34
if spring gateway is used
then the endpoint will be localhost:8765/orders/34

Get Request to fetch inventory by product name
Example: if the product name is "tables" as shown above
then endpoint will be localhost:8082/inventory-service/inventory/tables
if spring gateway is used
then the endpoint will be localhost:8765/inventory/tables

Important Note: if running the endpoint without Spring Gateway, then the endpoint will be localhost:8081/order-service/orders, if
    using Spring Gateway, then the endpoint will be localhost:8765/orders

JsonEntriesforTesting in Retry and DLQ scenario
{
  "productName": "fail-test",
  "price": 100.0,
  "quantity": 1,
  "status":"created"
}

if post request is made with the above JSON, it will fail and retry 2 times and then send the message to the dead letter topic (orders-dlt).
// The order will not be created in the inventory topic and the response will be as follows:
{
    "message": "Order failed to process in Inventory-Service after 2 retries, sent to dead letter topic",
    "order": {
        "orderId": 35,
        "productName": "fail-test",
        "quantity": 1,
        "price": 100.0,
        "status": "created"
    }
}
Note: create below topics in Kafka before running the test

inventory-topic
orders
orders-dlt

Make sure to run the Kafka server and create the topics using the commands from
kafka topics creation document.txt

Command to start/stop kafka server
----------------------------------

docker-compose up -d to start the server
docker-compose down to stop the server
